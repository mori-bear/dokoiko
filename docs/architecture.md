# Dokoiko アーキテクチャ設計メモ

## API分離トリガー条件

以下のいずれかが発生したら `server/` を活性化してAPI分離を検討する：

- [ ] 認証・認可ロジックが複雑化した
- [ ] 別クライアント（モバイルアプリ等）が必要になった
- [ ] フロントエンドとバックエンドのデプロイ頻度が乖離した
- [ ] パフォーマンス最適化でサーバーサイドの独立が必要になった

## 分離手順（メモ）

1. `src/lib/api/` をAPI仕様として整理
2. `server/` にAPIサーバーを実装
3. `src/lib/api/` のエンドポイントを `server/` に向け替える
4. デプロイパイプラインを分割
